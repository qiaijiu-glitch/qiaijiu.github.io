# 报告通用间距规范

## 通用间距 Token

| 语义层 | 用途 | 规范间距 |
| --- | --- | --- |
| `--report-space-2xs` | 超紧密装饰元素、双行小标签、极小型补白 | `4px` |
| `--report-space-xs` | 小型标签、数值徽标、标题内紧邻元素 | `6px` |
| `--report-space-sm` | 标题与正文、正文与辅助说明、小型内容组 | `8px` |
| `--report-space-md` | 卡片内容区块、模块标题与主体、常规卡片内边距 | `12px` |
| `--report-space-lg` | 同一模块内部的大组间距、弹层安全内边距 | `16px` |
| `--report-space-xl` | 页面主区块之间的垂直间距 | `24px` |
| `--report-space-2xl` | 特殊大留白场景，仅用于极少数页面级分段 | `32px` |

## 使用层级

- 页面主区块之间：统一使用 `--report-space-xl`
- 同一模块内部大组之间：统一使用 `--report-space-lg`
- 卡片内容区块之间：统一使用 `--report-space-md`
- 标题与正文、正文与辅助说明之间：优先使用 `--report-space-sm`
- 小型标签、数值、徽标等紧邻元素之间：优先使用 `--report-space-xs`
- 超紧密装饰元素或双行小标签：优先使用 `--report-space-2xs`

## 使用建议

- 报告页容器、模块容器、卡片容器优先用语义 token 控制 `gap` 和 `padding`
- 标题行到正文、正文到脚注、结论到建议等同层级关系保持同一 token
- 绝对定位图示、刻度 marker、图形几何位移不纳入通用 spacing 规范，可按视觉功能保留数值
- 新报告模块优先先定层级，再选 token，不要先写具体 `px`

## 禁止事项

- 报告内容区新增裸 `margin-top`、`gap`、`padding` 数值，优先走 spacing token
- 同一层级混用多个接近但无语义差别的数值，如 `10px`、`12px`、`13px`
- 用局部微调覆盖全局节奏，除非是图形定位、描边补偿、动画位移等功能性场景
- 将 spacing token 当成字号或尺寸 token 复用
