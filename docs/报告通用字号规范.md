# 报告通用字号规范

## 通用字号 Token

| 语义层 | 用途 | 规范字号 |
| --- | --- | --- |
| `--report-score-number` | 顶部总分数值，仅用于分数总览卡 | `29px` |
| `--report-score-unit` | 顶部分数单位“分” | `17px` |
| `--report-display-number` | 强调型大数值，如节段卡、详情主值 | `var(--fs-h-strong)` |
| `--report-page-title` | 子页标题、弹层主标题 | `var(--fs-h-page)` |
| `--report-card-number` | 普通数据卡主数值 | `var(--fs-h-value)` |
| `--report-card-unit` | 普通数据卡单位 | `var(--fs-h-unit)` |
| `--report-section-title` | 模块标题，如“人体成分概览”“调节建议” | `var(--fs-h-section)` |
| `--report-nav-label` | 切换栏、按钮文案、筛选项 | `var(--fs-body-nav)` |
| `--report-body` | 卡片正文、说明型主文案 | `var(--fs-body-info)` |
| `--report-body-secondary` | 次级说明、状态文案、辅助说明 | `var(--fs-body-secondary)` |
| `--report-label` | 卡片标题、小标签名 | `var(--fs-body-label)` |
| `--report-caption` | 极小型辅助文案、脚注、刻度 | `var(--fs-caption-medium)` |

## 使用场景

- 分数总览卡：只允许使用 `--report-score-number` 和 `--report-score-unit`
- 普通数据卡：标题用 `--report-label`，主值用 `--report-card-number`，单位用 `--report-card-unit`
- 调节建议、体态、围度、肩部的卡片说明：优先使用 `--report-body` / `--report-body-secondary`
- 模块标题与子页标题：分别使用 `--report-section-title` / `--report-page-title`
- Tabs、筛选项、切换按钮：统一使用 `--report-nav-label`
- Badge、脚注、图表浮标等小文案：优先使用 `--report-body-secondary` 或 `--report-caption`

## 禁止事项

- 报告内容区新增裸 `px` 字号
- 同一张卡片同时混用两个以上正文层级
- 用 `--report-section-title` 充当 `--report-card-number`
- 用旧的基础 token 直接跨级复用，绕过语义 token
